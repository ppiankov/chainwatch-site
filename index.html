Chainwatch

Runtime control plane for AI agent safety

AI agents need broad access to tools and data to be useful. But some actions cannot be undone — payments committed, credentials exposed, data destroyed, messages sent. Chainwatch enforces deterministic policy at these irreversible boundaries before the agent crosses them.

A single Go binary. No ML, no statistics, no prediction. Deterministic rules that block what must be blocked and allow what should pass.

An AI agent receives hazardous instructions. Chainwatch intercepts and blocks every one.

Install

go install github.com/ppiankov/chainwatch/cmd/chainwatch@latest

Or download a binary from the latest release (Linux and macOS, amd64 and arm64).

# macOS (Apple Silicon)
curl -sL https://github.com/ppiankov/chainwatch/releases/latest/download/chainwatch-darwin-arm64 -o chainwatch
chmod +x chainwatch
sudo mv chainwatch /usr/local/bin/

# Linux (amd64)
curl -sL https://github.com/ppiankov/chainwatch/releases/latest/download/chainwatch-linux-amd64 -o chainwatch
chmod +x chainwatch
sudo mv chainwatch /usr/local/bin/

How It Works

  • Wrap any command: chainwatch exec -- rm -rf /
  • Policy evaluates the action against denylist patterns and risk thresholds
  • Blocked commands are never executed (exit code 77)
  • Allowed commands run normally with an audit trail
Allow— action is safe, execute normally
Require Approval— operator must approve before execution
Deny— irreversible boundary, block unconditionally

Integration Modes

  • CLI wrapper — chainwatch exec -- <cmd>
  • gRPC server — chainwatch serve for multi-agent environments
  • HTTP proxy — chainwatch proxy intercepts agent HTTP traffic
  • LLM intercept — chainwatch intercept extracts tool calls from streaming responses
  • MCP server — chainwatch mcp for Claude Desktop

What It Blocks By Default

  • Destructive operations — rm -rf, dd if=/dev/zero, mkfs
  • Privilege escalation — sudo su, sudo -i, chmod 777
  • Code injection — curl | sh, wget | bash
  • Credential access — ~/.ssh/id_rsa, ~/.aws/credentials, .env
  • Self-targeting — attempts to kill, overwrite, or reconfigure chainwatch itself

All patterns are configurable via YAML. Built-in profiles available for common agent types: clawbot, devin, manus, soc-analyst, data-pipeline.

What Chainwatch Is Not

  • Not an ML-based anomaly detector — all decisions are deterministic
  • Not a logging or observability layer — enforcement, not detection
  • Not an LLM guardrail for prompt content
  • Not a permissions system — enforces boundaries, not roles

Performance

OperationLatency
Policy evaluation (allow)~5 us
Denylist match (block)~0.2 us
Denylist check (1000 patterns)~7 us
Audit log verify (10K entries)~30 ms

Zero-allocation denylist matching. Compiled regex patterns. Append-only audit log with SHA-256 hash chain.
